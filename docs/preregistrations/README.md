# Pre-registrations

Before any backtest or live-promotion gate, we write down the metric, the threshold, and the required sample size — then run it.
No moving goalposts after seeing results.
Each pre-registration is frozen (committed) before the experiment runs; the commit timestamp is the proof it came first.

Use `_template.md` for each experiment or gate.

## Why this is load-bearing

With ~104 matches in a single World Cup, a single tournament cannot statistically separate skill from luck (see the Phase 7 power analysis).
Cross-tournament history carries the statistical weight; the live tournament is confirmation, not proof.
Pre-registration is what stops us from retrofitting a threshold to whatever the result happened to be.
